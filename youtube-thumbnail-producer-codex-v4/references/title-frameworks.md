# Title Frameworks

Generate titles in three buckets. Use the user's exact keywords and angle whenever possible.

## Bucket 1: Search traffic

Goal: discoverability. Include core product, audience, action, and risk/benefit terms.

Patterns:

- [product] [audience] [action] tutorial | [benefit] | [risk/FAQ]
- [product] account opening guide | mainland users | fees / limits / verification / deposit
- no overseas account? how to use [product] for [goal]
- [year/version] [product] full guide | setup / registration / transfer / withdrawal

## Bucket 2: CTR hook

Goal: curiosity and click urgency. Shorter, more emotional, but still truthful.

Patterns:

- I tested [product]: is it still worth using?
- do not open [product] before watching this
- [product] changed again: mainland users should know this
- zero threshold or hidden costs? [product] explained
- the easiest way to [desired outcome]

## Bucket 3: Tutorial clarity

Goal: beginner trust. Direct and step-by-step.

Patterns:

- beginner guide to [product]: from registration to first use
- [product] setup in one video: account, verification, fees, transfer
- complete [product] tutorial for mainland users
- how to [action] with [product], step by step

## Selection rules

A good title has:

- one primary keyword
- one audience or use case
- one concrete benefit or concern
- no more than one strong emotional hook

When making thumbnail text, compress the title into 2-3 short phrases.
