## 1. Skill 是做什么的

`seedance-storyboard-prompt` 用于把小说、剧本、散文、粗分镜、场景设定、角色设定、PDF/TXT/DOCX 文本、以及视频生成后的问题反馈，转成适合 Seedance / 生视频模型使用的：

1. 分镜表格版
2. 最终可复制 Prompt 版
3. 垫图描述词（角色 / 石像 / 道具 / 场景）
4. 问题修正版 Prompt

## 2. 主要能力

### 2.1 小说 / 剧本转分镜
输入故事文本，输出：
- 镜头编号
- 时长
- 景别
- 角度
- 运镜
- 焦段
- 画面内容
- 中文对白
- 英文对白
- 声音
- 剪接点
- 最终可复制 Prompt

### 2.2 自动翻译人物台词
- 分镜表格：保留中文原句 + 英文翻译
- 最终 Prompt：默认只使用英文对白
- 如需中文对白，明确写：`保留中文台词`

### 2.3 垫图功能
可从分镜中提取：
- 角色
- 石像
- 道具
- 场景
- 关键画面

默认输出：
1. 场景档案
2. 英文 Prompt
3. Negative

### 2.4 问题修复
支持根据生成反馈修改 Prompt，例如：
- 人物变脸
- 年龄 / 服装漂移
- 字幕 / logo / 水印出现
- 多人同脸 / 双胞胎
- 石像不像石头
- 场景不稳定

## 3. 常用调用方式

### 3.1 生成前两个分镜
```text
使用 seedance-storyboard-prompt。
把下面原文生成前两个分镜。
输出：
1、分镜表格版
2、最终可复制 Prompt 版
人物对白自动翻译为英文。
不要字幕、不要配乐、不要 logo、水印，不要多人同脸。

原文：
【粘贴内容】
```

### 3.2 完整场次分镜
```text
使用 seedance-storyboard-prompt。
根据下面内容生成完整场次分镜。

场次编号：SC_x_x
画幅：竖屏9:16
风格：写实短剧
总时长：__s
镜头数：__镜
对白：中文自动翻译为英文
声音：不要配乐，仅允许同期对白、环境声、自然音效
输出：分镜表格版 + 最终可复制 Prompt 版

内容：
【粘贴内容】
```

### 3.3 垫图模式
```text
使用 seedance-storyboard-prompt 的垫图模式。

任务：从以下内容中提取一个垫图对象。
垫图对象：@____
画幅：16:9
用途：后续 Seedance 生视频参考图
输出格式：
1、场景档案
2、英文 Prompt
3、Negative

内容：
【粘贴分镜 / 原文】
```

### 3.4 问题修复
```text
使用 seedance-storyboard-prompt 修改 Prompt。

生成问题：
1、____
2、____
3、____

修改目标：
1、____
2、____
3、____

原 Prompt：
【粘贴 Prompt】
```

## 4. 最短触发方式
- `用 Seedance 分镜 Skill，把这段原文生成前两个分镜。`
- `用 Seedance 分镜 Skill，只给我最终可复制 Prompt。`
- `用垫图模式，提取 @老祭司石像。`
- `这个 Prompt 生成出来有问题，帮我按反馈修。`

## 5. 推荐工作流
```text
原文 / 剧本
↓
先提取关键人物、石像、道具、场景垫图
↓
用垫图生成参考图
↓
再根据原文生成视频分镜 Prompt
↓
用参考图 + 视频 Prompt 生成视频
↓
反馈问题
↓
修改 Prompt
↓
重新生成
```

## 7. 证据模式 / 依据反馈模式

当你不只想要优化后的 Prompt，而是想知道“为什么这么改、依据是什么、参考了哪些文件”时，使用证据模式。

推荐模板：

```text
使用 seedance-storyboard-prompt 的证据模式处理下面问题。

要求输出：
1、问题结论
2、参考文件名
3、文档内容摘要
4、当前分镜文本依据
5、判断标准
6、区分【文件明确依据】【当前分镜结构推断】【经验优化建议】
7、优化后的 Prompt

强制规则：
如果某条判断没有直接文件依据，请明确标注为“经验优化建议”或“当前分镜结构推断”，不要伪装成文档结论。

当前分镜：
【粘贴分镜】

当前问题：
【粘贴问题】
```

## 8. 新增资料时不要改 Skill

后续新增 PDF、DOCX、TXT、Markdown 或网页资料时，建议加入 Codex 项目资料库，而不是反复修改 Skill。

推荐项目结构：

```text
seedance-prompt-agent/
├── AGENTS.md
├── docs/
│   ├── MANIFEST.md
│   ├── sources/
│   ├── web/
│   └── cases/
├── scripts/
│   ├── ingest_file.py
│   ├── ingest_url.py
│   └── search_docs.py
└── skills/
    └── skill.zip
```

新增文件时说：

```text
把这个新文档加入 Seedance 知识库，更新 MANIFEST.md，后续回答要能引用它。
```

新增网页时说：

```text
把这个网页加入 Seedance 知识库，保存到 docs/web/，提取核心规则并更新 MANIFEST.md。
```


## 7. 白话使用方式：不用记模式名

你可以像和网页版 ChatGPT 对话一样使用，不需要说复杂命令。

### 加文档

```text
这个文档加进去，以后分析声音问题要参考。
```

智能体应该理解为：把文档加入资料库，提取摘要，更新索引，以后回答时引用。

### 加网页

```text
这个网页也加入资料库，以后生成 Seedance Prompt 要参考。
```

智能体应该理解为：读取网页正文，保存到网页资料区，提取规则，更新索引。

### 反馈画面问题

```text
这段生成出来声音怪，流泪也很生硬，帮我查资料后改。
```

智能体应该理解为：先查资料，再分析当前 Prompt，区分资料依据、当前内容依据、AI 推理和经验建议，最后给修复版 Prompt。

### 场次衔接问题

```text
SC_3_6 和 SC_3_7 接不上。上一镜是莉迪亚背影看美杜莎，下一镜不应该突然变成莉迪亚正对镜头。
```

智能体应该自己判断这是前后镜头空间关系和动作因果问题，不需要你说“空间连续性模式”。

### 垫图

```text
把老祭司石像提出来做垫图。
```

智能体应该输出：场景档案 + English Prompt + Negative。

核心原则：具体问题类型由智能体自己判断，不需要你不断添加规则。规则只约束它必须查资料、看上下文、说明依据、按你的格式输出。
