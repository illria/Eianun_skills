# 自然任务理解与自主判断工作流

作者：eianun

这个项目的目标不是让用户记复杂命令，而是让智能体像网页版 ChatGPT 一样理解自然语言需求。

用户可以直接给：

- 文档
- 网页
- 分镜
- Prompt
- 视频生成问题反馈
- 希望的输出格式
- 一句白话需求

智能体必须自己判断要做什么、查什么资料、输出什么格式。

## 核心工作流

1. 理解用户目标
   - 不要求用户说“证据模式”“垫图模式”“空间连续性问题”。
   - 根据上下文判断用户是要加资料、加网页、查资料、诊断问题、修改 Prompt、生成分镜、生成垫图，或其他相关任务。

2. 收集材料
   - 用户本次给的分镜 / Prompt / 反馈。
   - 用户上传的文件。
   - 用户给的网页。
   - `docs/MANIFEST.md`。
   - `docs/sources/`、`docs/web/`、`docs/cases/`。
   - `skills/skill.zip` 中的 Skill 规则，如适用。

3. 检索资料
   - 先看 MANIFEST。
   - 再查相关 docs。
   - 需要时用 `scripts/search_docs.py` 搜索关键词。
   - 新文件用 `scripts/ingest_file.py` 入库。
   - 新网页用 `scripts/ingest_url.py` 入库；若无法抓取，要求用户提供正文、PDF、DOCX 或截图。

4. 自主判断问题类型
   - 声音、口型、字幕、水印、人物一致性、镜头衔接、空间连续性、动作因果、时长、道具、垫图、Prompt 格式等都由智能体自己判断。
   - 不要只靠关键词。
   - 也允许判断为未预设的新问题。

5. 标明依据
   - 【资料明确依据】：来自 docs、网页、上传文件或知识库。
   - 【当前内容依据】：来自用户本次提供的分镜、Prompt、问题描述。
   - 【AI 推理判断】：基于资料和当前内容做出的分析。
   - 【经验优化建议】：没有直接资料依据，但符合视频生成经验的建议。

6. 按用户需求输出
   - 用户要 Prompt，就给修改后的 Prompt。
   - 用户要垫图，就给场景档案 + Prompt + Negative。
   - 用户要诊断，就给依据报告。
   - 用户只说“帮我修”，则输出简明诊断 + 修复版 Prompt。

## 白话输入示例

### 加资料

用户：

```text
这个文档加进去，以后分析声音问题要参考。
```

智能体应理解为：新增资料入库，提取摘要，更新 MANIFEST，说明后续用途。

### 加网页

用户：

```text
这个网页也加入资料库，以后生成 Seedance Prompt 要参考。
```

智能体应理解为：抓取网页，保存到 docs/web，提取规则，更新 MANIFEST。

### 反馈画面问题

用户：

```text
这段生成出来声音怪，流泪也很生硬，帮我查资料后改。
```

智能体应理解为：查资料，诊断声音和微表情问题，区分依据类型，给修复版 Prompt。

### 场次衔接问题

用户：

```text
SC_3_6 和 SC_3_7 接不上。上一镜是莉迪亚背影看美杜莎，下一镜不应该变成莉迪亚正对镜头。
```

智能体应理解为：比较前后场次，判断空间关系和镜头调度是否断裂，给出修复方案。

### 垫图

用户：

```text
把老祭司石像提出来做垫图。
```

智能体应理解为：提取目标，输出场景档案、英文 Prompt、Negative。

## 禁止行为

- 不要要求用户学习固定命令。
- 不要只有命中关键词才行动。
- 不要把所有具体问题都写死成规则。
- 不要只凭经验给结论。
- 不要把经验判断伪装成资料依据。
- 不要只给 Prompt，而不解释关键判断来源，除非用户明确只要最终 Prompt。
