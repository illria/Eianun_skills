# Bcut Subtitle Extractor

## Default behavior

For the user's normal request such as “提取最新必剪字幕生成 SRT”, do not ask which version to generate and do not choose between many modes. Run the bundled script with the newest project under `~/Movies/Bcut Drafts` and generate exactly two SRT files by default:

1. `extracted_subtitles.source.srt` — 普通版 / 源文件：保留必剪原始识别文本和原始对齐时间轴，不去标点、不去语气词、不合并。
2. `extracted_subtitles.enhanced.srt` — 增强版：保留普通版的字幕块密度和时间轴，只去除逗号/感叹号/句号和常见口语语气词；默认不智能合并，避免画面被长字幕铺满。

Default command:

```bash
python scripts/extract_bcut_subtitles.py --drafts-dir "~/Movies/Bcut Drafts"
```

If the current environment cannot access the user's local Mac path, ask the user to upload the relevant `.json`/`.bjson` draft file or the whole draft folder.

## Standard workflow

1. Run the default command against `~/Movies/Bcut Drafts`.
2. Return both generated SRT paths or downloadable files.
3. Only use `--list-projects` after extraction fails or the user says the newest project is not the desired one.
4. Only explain advanced parameters when the user asks to tune segmentation/cleanup.

Target a specific draft folder or file while still generating both versions:

```bash
python scripts/extract_bcut_subtitles.py \
  --project "/path/to/draft-or-file" \
  --output "/path/to/extracted_subtitles.enhanced.srt"
```

This will also write the ordinary source SRT next to the enhanced output as `.source.srt` unless `--enhanced-only` is explicitly used.

Inspect recent Bcut projects only for troubleshooting:

```bash
python scripts/extract_bcut_subtitles.py --drafts-dir "~/Movies/Bcut Drafts" --list-projects
```

If no captions are found, tell the user to search one known subtitle line:

```bash
grep -R -a -n "一条真实字幕内容" "~/Movies/Bcut Drafts"
```

Then rerun the script with `--project` pointing at the matched project folder or `.bjson/.json` file.

## Output format

Both outputs must be valid UTF-8 SRT:

```srt
1
00:00:00,000 --> 00:00:02,500
第一句字幕

2
00:00:02,500 --> 00:00:05,000
第二句字幕
```

Required formatting:

- Use sequential numeric indexes starting at 1.
- Use `HH:MM:SS,mmm --> HH:MM:SS,mmm` timestamps.
- Keep exactly one blank line between subtitle blocks.
- Save as UTF-8.

## Enhanced version rules

The enhanced SRT uses these cleanup rules by default:

- Preserve the source SRT block boundaries and timestamps. Do not merge subtitle screens by default.
- Remove selected punctuation from caption text: `,，.!！。`.
- Remove common口语语气词: `啊`, `嗯`, `哦`, `噢`, `喔`, `呃`, `额`, `诶`, `欸`, `哎`, `唉`, `呀`, `呐`, `呢`.
- Remove standalone English fillers such as `um`, `uh`, `er`, `ah`.
- Never remove commas from SRT timestamps.

Only when the user explicitly asks to repair split fragments, add `--smart-segment`. This mode is conservative and has hard visual caps so it does not create long subtitle walls:

```bash
--smart-segment --max-chars 22 --min-chars 6 --max-duration 3.2 --max-gap 0.35
```
