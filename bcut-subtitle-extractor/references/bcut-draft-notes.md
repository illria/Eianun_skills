# Bcut draft subtitle notes

Bcut/必剪 desktop drafts on macOS are commonly stored under:

```bash
~/Movies/Bcut Drafts
```

Important notes:

- The latest project is usually the most recently modified folder inside `Bcut Drafts`.
- Captions may be in `.json` or `.bjson` files rather than a ready-made `.srt` file.
- Text material often appears as a timeline segment with a `material_id` that points to a text material entry.
- Some text material stores the visible caption inside a JSON-encoded `content` string.
- Common timeline time units are microseconds or milliseconds; the bundled script normalizes both.

Manual fallback when extraction fails:

```bash
grep -R -a -n "replace-with-one-known-caption-line" "~/Movies/Bcut Drafts"
```

After locating the file containing captions, rerun the script with `--project` pointing to that folder or file.

## Clean subtitle behavior

The default extraction already creates an enhanced output for re-importing into editing software or using as a final subtitle track. `--clean` remains available as an explicit alias, but it is not needed for the normal workflow.

The enhanced output enables cleanup while preserving Bcut's original subtitle-screen density:

1. `--strip-punctuation`: remove comma, exclamation, and period punctuation.
2. `--strip-fillers`: remove common口语 filler words such as 啊/嗯/哦/呃/额/呢.

It does not enable `--smart-segment` by default. This is deliberate: merging too many neighboring subtitles makes the video frame fill with text.

The punctuation removal default is:

```text
,，.!！。
```

This deliberately removes only the punctuation the user requested, instead of deleting all symbols. Filler removal is also conservative; do not add words like `这个` or `那个` to `--filler-words` unless aggressive cleanup is wanted.

## Smart segmentation notes

Use `--smart-segment` only when the user explicitly wants to repair tiny split fragments. It will not perform real semantic NLP, but it uses reliable rules that help with common 必剪 ASR fragments. The mode has hard caps by default so it does not create subtitle walls:

```bash
--smart-segment --max-chars 22 --max-duration 3.2 --max-gap 0.35
```

- Do not split immediately after common connective or function words such as `的`, `了`, `和`, `在`, `把`, `被`, `给`, `因为`, `所以`, `但是`, `然后`, `以及`, unless merging would exceed the hard visual caps.
- Do not start the next subtitle screen with common particles such as `的`, `了`, `吗`, `呢`, `吧`, `啊`, `着`, `过`, `们`, unless merging would exceed the hard visual caps.
- Merge only across very small gaps by default (`--max-gap 0.35`) so a real pause usually becomes a new SRT block.
- Keep sentence punctuation available during segmentation, then remove punctuation afterward for the enhanced output.

If the result merges too much, stop using `--smart-segment` and return to the default two-file output.

## Source SRT notes

The source SRT is generated by default. It keeps the original extracted alignment before cleanup and smart segmentation, so the user can compare what 必剪 generated versus what the enhanced SRT changed.

Default paired output:

```bash
python scripts/extract_bcut_subtitles.py --drafts-dir "~/Movies/Bcut Drafts"
```

This writes:

```text
extracted_subtitles.source.srt     # 普通版 / original aligned source SRT
extracted_subtitles.enhanced.srt   # 增强版 / same timing density, cleaned text only
```
