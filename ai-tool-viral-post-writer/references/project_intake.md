# Project Intake Checklist

Use this checklist when a project/tool is provided.

## Minimum facts to gather

- Project name
- Official URL / GitHub repo
- One-line description
- Main user problem
- Target users
- Current pricing/free tier claims
- License if open-source
- Install or usage path
- Demo screenshots or README sections
- Current metrics if relevant: stars, forks, releases, last update
- Risks/limits: setup complexity, API keys, privacy, ToS, local hardware

## Question only when necessary

Do not ask many questions if the link can be inspected. Ask a follow-up only when:
- the user wants their own product positioned and the target audience is unclear
- the platform is not X/Twitter
- they want a specific persona/degree of hype different from default
- the provided claim is high-risk or unverifiable
