# Claim Safety Rules

## General factuality

Verify current facts when possible. If not verified, avoid exact numbers and hard claims.

Safer replacements:
- “整理了大量...” instead of “300 个” when not verified
- “免费额度/免费 tier” instead of “永久免费”
- “适合原型测试/学习” instead of “生产可直接替代”
- “目前 README 显示...” instead of absolute claims

## Free and pricing claims

Do not say:
- 永久免费
- 无限制
- 不用信用卡
- 零账单
- $0 替代 $200/月

Unless the official source confirms it now. Prefer:
- “有免费额度”
- “部分功能免费”
- “适合先做原型测试”
- “价格/额度可能变化，使用前看官方页面”

## GitHub metrics

Stars, forks, commits, last updated, trending status change often. Verify with GitHub connector or web before using. If old data is from user-provided reference, treat it as unverified until checked.

## Finance/trading content

If writing about stocks, crypto, prediction, trading, or profit claims:
- add “仅供研究/学习/娱乐，不构成投资建议”
- do not promise returns
- do not encourage copying strategies blindly
- avoid “红利/闭眼冲/稳赚”

## Scraping/API bypass/content access

If a tool accesses Twitter, Reddit, YouTube, GitHub, websites, or anti-detect browsers:
- focus on legitimate research/prototype use
- warn against spam, credential misuse, high-scale scraping, ToS violations
- avoid giving instructions for evasion, account abuse, or bypassing paid APIs at scale

## Security/leak/system prompt content

For leaks, jailbreaks, system prompts, credentials, or private data:
- avoid operational misuse instructions
- frame as security discussion if needed
- do not help exploit or bypass safeguards

## Benchmarks and model claims

Use benchmark numbers only when from official paper/README and clearly cite. Avoid “最强/吊打/遥遥领先” unless backed and still use caution.
